  ● Flask 和 Gunicorn 是不同层面的东西：

  ┌───────────┬──────────────────────────────────────────────┬──────────────────────────────┐
  │           │                    Flask                     │           Gunicorn           │
  ├───────────┼──────────────────────────────────────────────┼──────────────────────────────┤  │ 角色      │ Web 框架（应用逻辑）                         │ WSGI 服务器（HTTP 处理）     │
  ├───────────┼──────────────────────────────────────────────┼──────────────────────────────┤  │ 做什么    │ 路由、请求处理、ORM                          │ 多进程管理、请求排队、并发   │
  ├───────────┼──────────────────────────────────────────────┼──────────────────────────────┤
  │ HTTP 请求 │ app.run() 自带开发服务器，仅限开发用         │ 生产环境接收 HTTP 请求的入口 │
  ├───────────┼──────────────────────────────────────────────┼──────────────────────────────┤
  │ CLI 命令  │ flask --app app <command> 直接操作 Flask app │ 不参与                       │
  └───────────┴──────────────────────────────────────────────┴──────────────────────────────┘

  生产环境的请求链路：

  HTTP 请求 → Nginx(443) → Gunicorn(127.0.0.1:5000) → Flask app

  Nginx 处理 HTTPS/静态文件，Gunicorn 启动多个 worker 进程跑 Flask 应用，生产环境不会用 python app.py（那个是单线程开发服务器）。

  flask reset-admin-password 在生产环境依然可用：

  cd /opt/visitor-system/backend
  source venv/bin/activate
  export ENV=production
  flask --app app reset-admin-password

  这个 CLI 命令只是加载 Flask app 对象并调用函数，不走 HTTP 请求链路，与 Gunicorn 是否运行无关。它直接操作数据库（通过 app.app_context()），Gunicorn 管不着，也不需要经过 Nginx。
